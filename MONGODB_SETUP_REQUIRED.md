# 🔗 MongoDB Setup - What You Need to Do

## ✅ Quick Answer: YES, You May Need to Add Your IP

### The Issue
MongoDB Atlas has **IP Whitelist** - it only allows connections from whitelisted IP addresses for security.

### What You Need to Do

#### Step 1: Find Your Public IP
```bash
# Your public IP address:
curl ifconfig.me
```

#### Step 2: Add IP to MongoDB Atlas
1. **Go to:** https://cloud.mongodb.com
2. **Login** with your account
3. **Select** "karigar" project
4. **Go to:** Security → Network Access
5. **Click:** "Add IP Address"
6. **Choose one:**
   - **For Development:** "Current IP Address" (auto-detects your IP)
   - **For Testing:** `0.0.0.0/0` (allows anyone - only use for development!)
7. **Click:** Confirm
8. **Wait:** 2-3 minutes for changes to take effect

#### Step 3: Test Connection
```bash
# Run diagnostic
./check-mongodb-setup.sh

# Or start dev server
npm run dev

# Should show: ✓ Connected to MongoDB
```

## 🔐 Security Notes

### Do NOT Use in Production
```
❌ 0.0.0.0/0 (Allows anyone - security risk)
❌ Hard-code credentials in code
❌ Commit .env.local to git
```

### Do Use in Production
```
✅ Specific IP whitelist for your servers
✅ Strong passwords (auto-generated by MongoDB)
✅ Environment variables for credentials
✅ MongoDB Atlas backups enabled
✅ IP restrictions for database access
```
